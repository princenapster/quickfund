🔧 1. Core Concept: What is the System?
A lending web application is a platform where:
Users can apply for loans
Admins (or automated logic) review and approve loans
System tracks repayments, balances, interest, and history
It can be used for:
Microfinance
Employee loans
Peer-to-peer lending
Small business lending
School/organizational lending

The tech stack used in this project is a modern, high-performance web stack structured as follows:

### 1. Frontend
*   **Core Library:** **React** (v18.2.0) for building the user interface.
*   **Build Tool &amp; Bundler:** **Vite** (v5.2.0) for fast Hot Module Replacement (HMR) and optimized production builds.
*   **Styling:** **Vanilla CSS** with a robust custom design system (centralized HSL color variables, dynamic shadow/glow properties, hover states, and glassmorphism styling in `index.css`).
*   **Icons:** **Lucide React** for modern vector icons.
*   **Deployment Configuration:** Pre-configured for deployment via **Vercel** (`vercel.json`).

### 2. Backend API
*   **Web Server Framework:** **Fastify** (v4.26.2)—a high-performance, low-overhead Node.js web framework (modern alternative to Express).
*   **Runtime Environment:** **Node.js** with ES6 modules enabled (`"type": "module"`).
*   **Database Client:** **node-postgres (pg)** (v8.11.5) to manage PostgreSQL connection pooling, queries, and transactions.
*   **Environment Configuration:** **dotenv** to manage configuration variables (like port, database URLs, and API endpoints).

### 3. Database
*   **Database Engine:** **PostgreSQL**. The database setup is designed to automatically negotiate SSL connections when hosting on cloud databases (e.g., Supabase, Railway, Neon) while disabling it for local development (`localhost`).✅ Authentication
JWT (JSON Web Tokens)
